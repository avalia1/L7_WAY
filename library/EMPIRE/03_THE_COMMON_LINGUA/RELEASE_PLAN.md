## Submission Strategy (Tiered)

### Tier 1 — Top-Choice Conferences
1. **ACM SIGSOFT FSE (Foundations of Software Engineering)**
   - Best fit: bridges software architecture, type systems, self-adaptive systems
   - Format: 10-12 pages, ACM double-column
   - Deadline: Typically March/April for November conference

2. **IEEE/ACM ICSE (International Conference on Software Engineering)**
   - Flagship venue — NIER (New Ideas and Emerging Results) track
   - Format: 4 pages (NIER) or 10-12 pages (Technical)

### Tier 2 — Strong Alternatives
3. **ACM TOSEM (Transactions on Software Engineering and Methodology)**
   - Journal format allows full 25-30 page exposition
   - Rolling submissions

4. **IEEE TSE (Transactions on Software Engineering)**
   - Strong venue for formal architecture contributions

5. **ICSA (International Conference on Software Architecture)**
   - Purpose-built for architectural contributions — gateway pattern is direct fit

### Tier 3 — Domain-Specific
6. **ACM CHI** — if emphasizing responsive/intent-driven design paradigm
7. **IEEE VR / ACM ISMAR** — if leading with 777 spatial computing case study
8. **ACM Middleware** — if emphasizing gateway orchestration layer

## Pre-Submission Steps
1. Reformat from Markdown to ACM LaTeX template (acmart.cls)
2. Complete evaluation section with quantitative benchmarks
3. Prepare reproducibility package (L7 gateway + 777 table demo)
4. Identify 2-3 potential reviewers to suggest
5. File provisional patent BEFORE submission (publication creates prior art)

## Publication Timeline
- March 2026: LaTeX formatting and evaluation completion
- April 2026: Submit to FSE or ICSE NIER
- If rejected: August 2026 submit to TOSEM (journal, no deadline)
- Parallel: Post preprint to arXiv (cs.SE) to establish priority date
